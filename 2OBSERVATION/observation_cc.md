# Section 2 Observation — 写作框架（大纲版）

> **叙事主线**：`模态冲突机理` → `白盒成功的必要条件` → `黑盒可行性边界`
> **核心方法论**：严格区分**表征可读性**（Probe 能解码）与**决策因果性**（Patching 能翻转输出）

---

## 背景说明

- **表征可读性 ≠ 决策因果性**：Probe 可读只说明信息*存在*，Activation Patching 可控才说明信息被*采纳*。混淆二者会误导攻击策略。
- **架构关键特性**：不同 prompt 共享同一音频表征 $h = f_{\text{audio}}(x)$，因此 prompt 文本在中层引入的因果效应可覆盖音频扰动。
- **语音情绪攻击的特殊性**：(1) 语义-韵律在物理信号层深度耦合；(2) prompt 敏感性——纯音频模型不存在的问题维度。

---

## 2.1 冲突处理的机理（语义 vs 韵律）

**核心论点**：ALLM 内部呈现「表征解耦 + 决策仲裁」两阶段——早层韵律可读 ≠ 晚层决策采纳韵律。

### 段落逻辑流

1. **Opening**：语义-韵律冲突时模型依赖什么？→ 决定攻击策略的设计方向
2. **Probe 层级结构**（247 样本，197 冲突 / 50 一致）
   - 早层 0-14：韵律主导，$D(\ell)>0$，均值 $\approx 0.146$，峰值 Layer 5 $D \approx 0.2146$，$Acc_{\text{prosody}}(0) \approx 0.842$
   - 中层 14-23：融合态，$D \approx 0$，Layer 14-15 符号翻转
   - 晚层 23-35：语义占优，$Acc_{\text{semantic}}(27) \approx 0.830$，Layer 26 $D \approx -0.0414$
   - ⚠️ Layer 29-34 韵律回弹，可能为残差连接伪影，需谨慎解读
3. **Logit Lens 决策轨迹**（197 冲突样本，5-way restricted）
   - $\text{margin}(\ell) = \text{logit}_{\text{prosody}} - \text{logit}_{\text{semantic}}$
   - Layer 0-22：margin $\approx 0$，大量 "other" 输出，决策未形成
   - **Layer 23 起**：margin 显著转负，语义 win-rate 持续上升
   - **关键洞察**：早层韵律*表征*优势不转化为决策偏好 → 表征可读性 ≠ 决策采纳性
4. **Activation Patching 因果确认**
   - 语义 patch（Layer 0-12）：Flip-to-Target $\approx 0.65$，Delta Logit 5-6 → **强定向控制**
   - 韵律 patch（峰值 ~Layer 9）：Flip-to-Target $\approx 0.14\text{-}0.26$，Delta Logit $\approx 2.2\text{-}2.3$ → **弱扰动效应**
   - **Layer 14-15**：audio span 可控性边界，此后信息扩散至全局位置
5. **小结**：Probe（what is encoded）→ Logit Lens（what drives output）→ Patching（what causally determines output），三者共同确认「语义优先仲裁策略」
6. **过渡**：既然决策偏向语义，白盒音频扰动为何仍能达 ~80%？→ 引出 2.2

### 缺口分析（2.1）

#### A. 逻辑链断裂

1. **Layer 14-15 vs Layer 23 的边界落差未解释**
   - Probe 与 Patching 均指向 Layer 14-15 为关键边界，但 Logit Lens 的决策转折在 Layer 23，中间差 8 层。当前框架将两者并列呈现但未解释这一落差。
   - 需要回答：是"表征转变先于决策转变"（信息结构先变，决策惯性滞后）？还是方法论差异（Logit Lens 在中间层本身不可靠，Layer 23 只是"可靠读出"的起点而非真实转折）？
   - **补充方案**：在 Layer 14-23 区间做更细粒度的 Patching（逐层 + 逐位置），区分"信息已转变但决策未跟进"与"Logit Lens 中间层噪声"两种解释。

2. **"信息扩散至全局位置"缺乏直接证据**
   - Patching 在 Layer 14-15 后失效，当前解释是"信息通过 attention 扩散到 text token 位置，局部 audio span patch 不再有效"。但这只是一种解释，未被直接验证。
   - 替代解释：信息并未扩散，而是被 MLP 层非线性变换消解了。
   - **补充方案**：做 position-level patching——分别 patch audio span / text span / 随机 span，验证 Layer 15+ 时 text span 的 patch 是否获得了原本属于 audio span 的控制力（信息迁移的直接证据）。

3. **Probe → Logit Lens → Patching 的"证据升级"叙事隐含假设**
   - 当前叙事是"从相关性到因果性的递进"，但三种方法测量的对象不完全相同：Probe 测的是 audio span 池化后的表征，Logit Lens 测的是最后一个 token 位置的投影，Patching 测的是 audio span 的干预效应。三者的"层"含义一致，但"位置"不一致。
   - **补充方案**：明确说明三种方法的位置选择差异，并论证为什么这些差异不影响核心结论（或承认这是一个局限）。

#### B. 实验空白

4. **缺少三组对照的对齐呈现**
   - 当前 Probe/Logit Lens/Patching 实验主要基于 197 条冲突样本。50 条一致样本（Audio-only / Consistent）的对照结果未系统报告。
   - 如果一致样本上也出现类似的层级结构，则"层级解耦"是模型的通用处理模式而非冲突特异性现象——这会改变结论的解读。
   - **补充方案**：对三组条件（Audio-only / Conflict / Consistent）分别报告 Probe $D(\ell)$、Logit Lens margin、Patching Flip-to-Target，对齐呈现。

5. **缺少 Attention Pattern 分析**
   - 2.1 讨论了信息在各层的编码和因果效应，但未分析 attention 机制如何在层间传递信息。Attention pattern 可以揭示：(a) audio token 之间的自注意力何时减弱；(b) text token 对 audio token 的交叉注意力何时增强；(c) 信息从 audio 位置"流向"text 位置的具体层。
   - 这直接补充"Layer 14-15 信息扩散"的机制解释。
   - **补充方案**：计算各层 attention 权重矩阵中 audio→audio、audio→text、text→audio 三类子块的平均权重，绘制随层变化曲线。

6. **跨模型泛化性完全缺失**
   - 所有机理实验基于 OpenS2S（35 层）。Kimi-Audio（28 层）的层级结构是否一致？如果 28 层模型的边界在不同位置（如 Layer 10-11 和 Layer 18），则 2.1 的结论需要参数化表述（"边界在 ~40%-65% 深度处"）而非绝对层号。
   - **补充方案**：至少在 Kimi-Audio 上复现 Probe 层级曲线，验证定性结论是否成立。

#### C. 未排除的替代假说

7. **线性 Probe 的表达力限制**
   - 线性 Probe 检测不到韵律信息 ≠ 韵律信息不存在。晚层韵律信息可能以非线性方式编码（如分布在多个维度的交互项中），线性 Probe 无法捕获。
   - 如果用 MLP Probe（2 层）在晚层也能解码韵律，则"晚层语义主导"的结论需要修正为"晚层语义在*线性可读性*上主导"。
   - **补充方案**：增加 MLP Probe 对照（2 层 ReLU），对比线性 vs 非线性 Probe 在各层的准确率差异。如果差异在晚层显著增大，说明韵律信息以非线性形式存在。

8. **Logit Lens 的词表偏差可能伪造 Layer 23 转折**
   - 如果 5 个情绪词在词表中的先验概率不均（如 "happy" 的 unigram 频率远高于 "angry"），Logit Lens 的 margin 可能反映词频偏差而非真实决策偏好。
   - **补充方案**：(a) 报告 5 个情绪词的 unigram log-probability 基线；(b) 用 margin - baseline 的校正值重新绘制曲线；(c) 验证 Layer 23 转折是否在校正后仍然成立。

9. **晚层韵律回弹（Layer 29-34）的三种竞争解释**
   - 当前仅提及"残差连接伪影"一种解释。至少还有两种：(a) 模型在生成前需要"回忆"韵律信息用于语音合成（OpenS2S 是语音到语音模型）；(b) 统计噪声（样本量有限导致的随机波动）。
   - **补充方案**：(a) 检查回弹幅度是否在 bootstrap CI 内显著；(b) 对比语音生成任务 vs 纯文本生成任务下的晚层韵律 Probe，区分"合成需求"与"伪影"。

## 2.2 白盒攻击成功的原因

**核心论点**：训练 prompt 下 ~80% 成功率依赖三条件同时满足；跨 prompt 失效（40%/50%/20%）的根因是中层文本因果主导性增强。

### 段落逻辑流

1. **Opening — 反推逻辑**：不直接解释"为什么成功"，而是从"跨 prompt 为什么失败"反推"成功时什么条件被满足了"
2. **现象呈现**：跨 prompt 鲁棒性数据

   | Prompt 类型 | 成功率 | 特征 |
   |------------|--------|------|
   | 训练 prompt | ~80% | 基线 |
   | 直接分类 | ~40% | 中-高文本覆盖 |
   | 描述+分类 | ~50% | 中等文本覆盖 |
   | CoT 类 | ~20% | 多步文本链，覆盖最强 |

3. **三条件模型**（从 2.1 机理推导）：
   - **C1 早层可注入**：扰动能在 Layer 0-14 改变韵律表征（2.1 Probe 证据支撑，早层韵律可读性高 → 扰动有作用空间）
   - **C2 中层可存活**：扰动效果能穿越 Layer 14-23 的融合竞争区而不被消解（2.1 Patching 证据：Layer 14-15 为可控性边界）
   - **C3 晚层未被覆盖**：在 Layer 23-35 的仲裁阶段，扰动信号未被文本指令的结构性效应覆盖
4. **失效机制分析**：
   - Prompt-Audio 冲突实验（18 音频 × 3 条件）：文本指令在 **Layer 5-20 保持强因果效应**，音频信号在此被 wash-out
   - 仲裁固化窗口：**Layer 26-28**，此后决策不可逆
   - CoT prompt 失效最严重（~20%）：多步推理链在中层产生更强的文本因果主导，C3 条件被彻底破坏
   - 训练 prompt 成功（~80%）：该 prompt 的文本结构恰好在中层因果效应较弱，C2/C3 条件得以满足
5. **小结**：白盒成功 = C1 ∧ C2 ∧ C3；跨 prompt 失效的根因是不同 prompt 在中层的文本覆盖强度不同，破坏了 C2 或 C3
6. **过渡**：三条件中哪些在黑盒场景下可被满足？→ 引出 2.3

### 缺口分析（2.2）

#### A. 逻辑链断裂

1. **三条件模型是"推导"而非"验证"**
   - C1/C2/C3 从 2.1 机理合理推导而来，但目前没有任何实验直接验证每个条件的*必要性*。"合理推导"在顶会审稿中不等于"已证实"——reviewer 会问：如果破坏 C1 但保留 C2/C3，成功率是否真的下降？
   - **补充方案**：设计三组消融实验，每组仅破坏一个条件：
     - 破坏 C1：在 Layer 0-14 将扰动后的 audio 表征恢复为原始表征（early-layer restoration），观察成功率是否从 ~80% 显著下降
     - 破坏 C2：在 Layer 14-23 注入额外的文本覆盖信号（mid-layer text amplification），观察成功率变化
     - 破坏 C3：在 Layer 23-35 用强文本指令替换原始 prompt 表征（late-layer override），观察成功率变化
   - 三组消融的成功率均显著下降 → C1∧C2∧C3 的必要性得到验证

2. **C1→C2→C3 是"同时满足"还是"顺序依赖"？**
   - 当前表述为 C1∧C2∧C3（逻辑与），隐含三者独立。但实际上可能存在顺序依赖：C1 失败（早层注入不足）时，C2 和 C3 的讨论无意义（没有信号可以"存活"或"被覆盖"）。
   - 如果是顺序依赖，则失败模式分析应按优先级排序：先检查 C1 → 若 C1 满足再检查 C2 → 最后检查 C3。
   - **补充方案**：在失败案例分析中，按 C1→C2→C3 顺序做逐层诊断，统计失败样本中各条件的失败占比（如 C1 失败占 15%、C2 失败占 50%、C3 失败占 35%），建立失败归因的层级结构。

3. **"描述+分类"成功率（~50%）高于"直接分类"（~40%）的反常现象未解释**
   - 按直觉，更长/更复杂的 prompt 应产生更强的中层文本覆盖，成功率应更低。但描述+分类（更长）反而比直接分类（更短）成功率高。
   - 可能解释：描述任务引导模型更多"关注"音频内容（增强 audio attention），反而为扰动信号提供了传播通道；或者直接分类 prompt 的特定措辞恰好在中层产生了更强的情绪先验覆盖。
   - **补充方案**：对这两类 prompt 分别做中层 attention 分析（text→audio vs audio→audio 权重对比），验证"描述任务增强音频注意力"假说。

4. **训练 prompt 的"中层覆盖弱"是偶然还是被优化利用？**
   - 白盒攻击在训练 prompt 下优化损失函数，梯度信号可能隐式地"选择"了中层覆盖较弱的扰动路径。如果是这样，~80% 成功率不是因为训练 prompt 天然覆盖弱，而是因为优化过程找到了规避覆盖的扰动模式。
   - 这一区分对 2.3 黑盒推导至关重要：如果是"prompt 天然弱"，黑盒攻击者只需找到类似的弱 prompt；如果是"优化利用"，黑盒攻击者需要替代梯度的优化能力。
   - **补充方案**：(a) 对训练 prompt 做中层文本覆盖强度量化，与其他 prompt 对比；(b) 分析白盒扰动在各层的梯度分布，检查梯度是否集中在中层覆盖较弱的子空间。

#### B. 实验空白

5. **"中层文本覆盖强度"缺乏可计算的量化指标**
   - 表 B 中的"低/中/高"是定性判断，无法支撑回归分析或统计检验。需要定义一个标量指标。
   - **补充方案**：定义 Text-Dominance Index (TDI)：在 Layer 5-20 对 text token 做 zero-out 或 mean-ablation，测量输出变化幅度（如 KL 散度或 logit shift）。TDI 越高 → 文本覆盖越强。然后做 TDI vs 攻击成功率的散点图 + 相关系数。

6. **成功率数据的样本量与统计功效不足**
   - ~80%/40%/50%/20% 基于多少样本？CONTEXT.md 提到白盒攻击初步结果基于 10 条样本。10 条样本的二项分布 95% CI 极宽（如 80% 的 CI 为 [44%, 97%]），四类 prompt 之间的差异可能不显著。
   - **补充方案**：(a) 扩大样本量至至少 50-100 条；(b) 报告 exact binomial CI；(c) 做 prompt 类型间的 Fisher exact test 或 χ² 检验，确认差异的统计显著性。

7. **失败案例的决策路径追踪缺乏具体方案**
   - "待补充验证"中列出了"失败案例决策路径追踪"，但未说明具体如何操作。
   - **补充方案**：对每个跨 prompt 失败样本，逐层计算：(a) 扰动后 vs 原始的 margin 差（扰动信号强度）；(b) 扰动后 vs 原始的 Flip rate（扰动定向性）。绘制"扰动信号衰减曲线"，标注信号在哪一层降至噪声水平。将失败样本按衰减模式聚类为：Early 注入不足型 / Mid wash-out 型 / Late 仲裁反转型。

8. **因果反证实验（中层文本抑制）的设计需要细化**
   - "在 Layer 5-20 抑制文本 token 表征"的操作不明确：是 zero-out？mean-ablation？还是替换为中性指令的表征？不同操作的含义不同。
   - **补充方案**：设计三级干预：(a) text token zero-out（完全移除文本信号）；(b) 替换为中性指令表征（如"请分析这段音频"→ 移除情绪相关指令但保留任务框架）；(c) 替换为训练 prompt 表征（验证是否能恢复到 ~80% 成功率）。三级干预的成功率梯度可以量化文本覆盖的"剂量-效应"关系。

#### C. 未排除的替代假说

9. **跨 prompt 失效可能不是"中层覆盖"，而是"优化目标不匹配"**
   - 白盒攻击的 $\mathcal{L}_{\text{emo}}$ 在训练 prompt 下优化，扰动被调整为在该 prompt 的特定 token 位置产生最大效果。换一个 prompt 后，token 序列变化导致 attention pattern 完全不同，扰动效果自然下降——这是"优化过拟合"而非"中层覆盖"。
   - 两种解释的区分：如果是"优化过拟合"，在训练 prompt 上用随机扰动（非优化）的成功率应远低于 80%，且与其他 prompt 的差距应缩小；如果是"中层覆盖"，即使用随机扰动，训练 prompt 与其他 prompt 的成功率差距仍应存在。
   - **补充方案**：增加"随机扰动基线"——在所有 prompt 类型上测试等强度随机扰动的成功率，作为"无优化"对照。

10. **C2"中层存活"与 C3"晚层未覆盖"可能是同一机制的不同表现**
    - 中层文本覆盖（破坏 C2）和晚层文本覆盖（破坏 C3）可能不是两个独立条件，而是同一个"文本因果主导性"在不同层的连续表现。如果是这样，三条件模型应简化为两条件（C1 + C2/C3 合并）。
    - **补充方案**：在消融实验中检查 C2 和 C3 的独立性——如果破坏 C2 时 C3 也自动被破坏（或反之），则两者不独立，应合并论述。

## 2.3 声音黑盒处理的条件

**核心论点**：黑盒可行性取决于 2.2 三条件中哪些可被外部查询能力替代；可行域由四维约束（输入粒度、反馈类型、查询预算、稳定性）共同决定。

### 段落逻辑流

1. **Opening**：从 2.2 三条件出发，逐项分析黑盒场景下的可满足性
2. **四维约束推导**：

   | 维度 | 白盒条件 | 黑盒替代 | 最低要求 | 失效边界 |
   |-----|---------|---------|---------|---------|
   | 输入粒度 | 梯度级波形优化 | 波形级输入可控 | 可提交 waveform | 仅文本输入 |
   | 反馈类型 | 完整梯度 | 概率/logit 分布查询 | 概率反馈 | 仅 top-1 + 严格限频 |
   | 查询预算 | 无限制（白盒单次优化） | 迁移先验 + 低方差梯度估计 | 中等预算（$10^2$-$10^3$ 量级） | 严格预算 + 高噪声反馈 |
   | 稳定性 | 训练 prompt 固定 | prompt-ensemble 一致性 | 可切换 prompt 模板 | 固定强指令模板（中层覆盖不可规避） |

3. **核心风险**：中层语义覆盖（2.2 的 C2/C3 失效机制）在黑盒下更难规避——攻击者无法观测中间层状态，只能通过多 prompt 查询间接评估稳定性
4. **可行域边界**：
   - **可行区**：波形可控 + 概率反馈 + 中等预算 + prompt 可切换 → 基于迁移性的查询高效攻击
   - **灰色区**：仅 top-1 反馈 / 有限预算 → 需要强迁移先验，成功率显著下降
   - **不可行区**：仅文本输入 / 固定强指令模板 / 极低预算 → 中层覆盖无法规避
5. **小结**：建立白盒发现到黑盒威胁模型的完整映射，明确 Section 4 方法论需要解决的核心问题——如何在有限查询预算下规避中层语义覆盖

### 待补充验证

| 验证项 | 内容 | 优先级 |
|-------|------|-------|
| 跨 prompt 稳定性量化 | 多模板/多 seed/重采样条件下攻击效果方差 | 高 |
| 查询复杂度分析 | success@k queries 曲线，确定最低有效查询量 | 高 |
| 迁移效率评估 | surrogate → target 的 win-rate 与 query-saving | 中 |
| top-1 反馈失效阈值 | 仅 top-1 条件下成功率随查询预算的衰减曲线 | 中 |

### 缺口分析（2.3）

#### A. 逻辑链断裂

1. **四维约束的推导缺乏形式化映射**
   - 从 C1/C2/C3 到"输入粒度 / 反馈类型 / 查询预算 / 稳定性"四维的映射是直觉性的，缺乏严格的对应关系论证。具体地：
     - C1（早层可注入）→ 输入粒度：逻辑清晰（需要波形级访问才能注入扰动）
     - C2（中层可存活）→ 反馈类型 + 稳定性：映射模糊——为什么"中层存活"需要"概率反馈"？黑盒攻击者无法观测中间层，概率反馈只能间接反映最终输出，与"中层存活"之间缺少推理桥梁
     - C3（晚层未覆盖）→ 稳定性：映射合理但未量化——"prompt-ensemble 一致性"如何量化"晚层未覆盖"的程度？
   - **补充方案**：为每条映射写出显式推理链，形如"C2 要求扰动在中层存活 → 黑盒下无法直接观测中层 → 需要通过输出概率的变化间接推断扰动是否存活 → 因此需要概率级反馈而非仅 top-1"。

2. **"可行区 / 灰色区 / 不可行区"的边界缺乏判据**
   - 当前三区划分是定性的。什么成功率阈值算"可行"？>50%？>30%？不同应用场景（安全评估 vs 实际攻击）的阈值不同。
   - **补充方案**：(a) 定义可行性阈值（如 success rate > 30% 且 query budget < $10^3$ 为"可行"）；(b) 在相图中用等高线标注阈值边界而非离散分区。

3. **2.2→2.3 的推导跳过了"梯度替代"这一核心问题**
   - 白盒攻击依赖梯度优化，黑盒攻击的核心挑战是如何替代梯度。当前四维约束中"查询预算"隐含了梯度估计的需求，但未显式讨论梯度估计方法（NES / SPSA / finite-difference）的精度与查询开销之间的 trade-off。
   - **补充方案**：增加一个子段落，讨论"梯度替代策略"：(a) 零阶梯度估计（query-based）；(b) 迁移梯度（surrogate-based）；(c) 两者混合。明确每种策略对反馈类型和查询预算的具体要求。

#### B. 实验空白

4. **迁移性假设完全缺乏实证**
   - 查询预算维度的核心假设是"迁移先验可以降低查询量"，但目前没有任何迁移性实验数据。如果 OpenS2S 上的扰动对 Kimi-Audio 的迁移率极低（如 <10%），则"迁移先验"假设不成立，查询预算估计需要大幅上调。
   - **补充方案**：做最小可行迁移实验——将 OpenS2S 白盒扰动直接应用于 Kimi-Audio，测量 zero-shot transfer rate。即使只有 10 条样本，也能给出迁移性的量级估计。

5. **"概率反馈"的现实可得性未讨论**
   - 四维约束假设攻击者可以获得概率/logit 分布，但大多数商业语音 API（如 GPT-4o 语音模式、Gemini 语音）只返回文本，不提供 logit 输出。如果概率反馈在现实中不可得，则"可行区"大幅缩小。
   - **补充方案**：(a) 调研主流 ALLM API 的反馈类型（文本 / top-k / 概率分布）；(b) 增加"仅文本反馈"条件下的可行性分析——此时攻击退化为 decision-based attack，查询复杂度可能上升 1-2 个数量级。

6. **查询预算 $10^2$-$10^3$ 的估计缺乏依据**
   - 这一量级是类比图像黑盒攻击文献的经验值，但语音信号的维度远高于图像 patch（一段 5 秒 16kHz 音频 = 80,000 维），梯度估计的方差与输入维度正相关。
   - **补充方案**：(a) 给出理论下界：基于零阶优化的收敛速率，$O(d/\epsilon^2)$ 次查询（$d$ 为输入维度）；(b) 讨论降维策略（如在 mel-spectrogram 空间或 Audio Encoder 输出空间优化）对查询量的影响。

7. **缺少 prompt-ensemble 策略的具体设计与验证**
   - "稳定性"维度提到"prompt-ensemble 一致性"作为黑盒下评估中层覆盖的手段，但未给出具体方案：用哪些 prompt？如何聚合多 prompt 的反馈？一致性的判据是什么？
   - **补充方案**：设计 prompt ensemble 协议——(a) 选择 3-5 个覆盖强度不同的 prompt（基于 2.2 的 TDI 排序）；(b) 对每个扰动候选，在所有 prompt 上查询；(c) 以"多数 prompt 成功"作为稳定性判据。验证该协议是否能筛选出跨 prompt 鲁棒的扰动。

#### C. 未排除的替代假说

8. **黑盒攻击可能不需要"满足白盒三条件"**
   - 当前 2.3 的整个推导建立在"黑盒攻击需要替代白盒三条件"的前提上。但黑盒攻击可能走完全不同的路径——例如，不通过韵律扰动，而是通过对抗性语义注入（在语音中嵌入对抗性语义内容）来直接利用模型的"语义优先"策略。
   - 如果存在绕过三条件的攻击路径，则 2.3 的可行域分析是不完备的（低估了威胁）。
   - **补充方案**：(a) 在 2.3 末尾明确声明"本分析限于韵律扰动路径的黑盒化，不排除其他攻击向量"；(b) 简要讨论语义注入路径的可能性与威胁等级。

9. **跨模型黑盒场景的特殊性未讨论**
   - 黑盒攻击的一个重要场景是跨模型迁移：攻击者在 surrogate model（如开源的 OpenS2S）上生成扰动，迁移攻击 target model（如闭源的 GPT-4o）。这一场景下，2.1 的机理发现（层级结构、边界位置）是否跨模型成立，直接决定迁移攻击的可行性。
   - 如果不同模型的层级边界位置差异大（如 OpenS2S 在 Layer 14-15，Kimi-Audio 在 Layer 10-11），则基于 OpenS2S 机理设计的扰动可能在 Kimi-Audio 上失效。
   - **补充方案**：(a) 讨论"机理迁移性"与"扰动迁移性"的区别；(b) 提出跨模型验证的最小实验方案（至少在两个模型上对比 Probe 层级曲线的定性一致性）。

10. **防御视角的对偶分析缺失**
    - 2.3 纯粹从攻击者视角分析可行性，但 Observation 章节如果能同时指出防御启示，会显著增强论文的贡献度和审稿人好感。
    - 具体地：2.1 发现"中层文本覆盖可以压制音频扰动"→ 防御者可以通过设计"强覆盖 prompt"（类似 CoT 的多步指令）来提高鲁棒性；2.2 发现"C2 是最脆弱的条件"→ 防御者应重点加固中层的文本-音频竞争。
    - **补充方案**：在 2.3 末尾增加一段"防御启示"（1-2 句），为 Section 7 Discussion 埋下伏笔，但不展开（避免抢 Discussion 的内容）。

## 关键要点

- **层级解耦假说**：ALLM 对语义-韵律冲突的处理呈现清晰的层级分离——早层韵律表征主导，晚层语义决策主导，14-15 层和 23 层是两个关键边界
- **表征-决策分离**：表征可读性（Probe）与决策因果性（Patching）系统性不一致，韵律"可读"但不"被采纳"
- **文本主导仲裁**：文本指令在中层（5-20）保持强因果效应，是跨 prompt 攻击失效的根因
- **白盒三条件模型**：成功 = 早层可注入 ∧ 中层可存活 ∧ 晚层未被覆盖
- **黑盒可行域**：由输入粒度 × 反馈类型 × 查询预算 × 稳定性四维约束界定，中层语义覆盖是首要风险

## 对比分析

### 表 A：三种机制工具在层级边界识别上的一致性

| 方法 | 测量对象 | 可回答问题 | 边界识别 | 局限 |
|-----|---------|-----------|---------|------|
| Probe | 表征可读性 | 信息在哪层可读 | Layer 14-15（$D$ 符号翻转） | 非因果，可能过拟合 |
| Logit Lens | 决策轨迹近似 | 哪层开始偏向语义决策 | Layer 23（margin 转负） | 词表敏感，中间层不稳定 |
| Activation Patching | 因果控制力 | 哪层干预能翻转输出 | Layer 14-15（可控性边界） | 依赖 patch 构造方案 |

**一致性**：Probe 与 Patching 在 Layer 14-15 边界上高度一致；Logit Lens 的 Layer 23 转折点与 Probe 的晚层语义占优区间吻合。
**差异**：Probe 显示晚层韵律回弹，但 Logit Lens 和 Patching 均不支持韵律在决策层面的复苏 → 回弹可能是表征层伪影。

### 表 B：Prompt 类型与攻击效果关联

| Prompt 类型 | 成功率 | 中层文本覆盖强度 | C2 满足度 | C3 满足度 |
|------------|--------|----------------|----------|----------|
| 训练 prompt | ~80% | 低 | 高 | 高 |
| 直接分类 | ~40% | 中-高 | 中 | 低 |
| 描述+分类 | ~50% | 中 | 中 | 中 |
| CoT 类 | ~20% | 高 | 低 | 极低 |

## 注意事项

- ❌ 不要将「Probe 韵律占优」等同于「模型依赖韵律决策」——这正是表征-决策分离的核心
- ❌ 不要将「训练 prompt 80% 成功率」外推为「跨 prompt 鲁棒」——30-60% 的跌幅说明条件依赖性
- ❌ 不要孤立报告 Flip-to-Target 而忽略 Delta Logit——前者是定向性，后者是强度，需同时达到显著
- ✅ 2.2 的论证核心是「必要条件反推」，不是重复 2.1 的数据
- ✅ 2.3 必须同时写清「可行域」与「失效域」，避免过度乐观的威胁评估
- ✅ 所有"推测"性结论（如晚层韵律回弹的解释）必须明确标注为假说
